Figma Make 用プロンプト
Design a minimal, distraction-free web application UI called “small-goal”.

Concept:
Users can set only one important goal per day. The UI should encourage deep thinking and focus rather than task accumulation.

Design style:

ultra minimal productivity UI

calm, thoughtful atmosphere

lots of whitespace

typography-focused design

Apple Notes / Linear / Notion style simplicity

soft neutral colors (white, light gray, subtle accent color)

Typography:

large readable headline

system font or modern sans-serif

emphasize clarity and calmness

Layout:
centered single-column layout
max width: 600px

Screen 1 — Goal Input
Headline:

“今日の重要なゴールは一つだけ”

Input field:

Placeholder:

“例：PRDを書く”

Constraint text:

“30文字以内”

Primary button:

“次へ”

Design note:

very large input box

encourage focus

nothing else on screen

Screen 2 — Reason Input
Headline:

“なぜこのゴールを達成する必要がありますか？”

Sub text:

“140文字以内で書いてください”

Text area placeholder:

“このゴールが重要な理由を書いてください”

Below the text area show subtle guidance text:

“If writing this makes you feel the goal is not important, discard it and set a better one.”

Primary button:

“ゴールを確定”

Secondary link:

“別のゴールを設定する”

Screen 3 — Today's Goal
Large centered card.

Title:

“Today's Goal”

Goal text displayed very large.

Below it show:

“Why this matters”

Display the reason text.

Primary button:

“完了”

Minimal UI.

Screen 4 — Completion State
Celebration but minimal.

Large message:

“🎉 今日の重要な仕事を完了しました”

Below show streak information:

“3日連続達成”

Button:

“履歴を見る”

Screen 5 — Goal History
Simple timeline list.

Each item:

Date
Goal
Status icon

Example:

2026-03-09
PRDを書く
✓ 完了

2026-03-08
論文を読む
✗ 未完了

Design tone
The product should feel like:

a thinking tool

not a productivity dashboard

quiet

reflective

intentional

Avoid:

dashboards

statistics overload

many buttons

complex navigation

The entire app should feel like a daily ritual.
